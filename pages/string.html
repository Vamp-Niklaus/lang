std::string (and Equivalents)

Deep dive into Strings across languages, focusing on immutability, manipulation, and special features.

std::string (Mutable)

In C++, std::string is heavily optimized and mutable. You can change individual characters directly via indexing.

#include <iostream>
#include <string>
#include <algorithm>

std::string s = "Hello World";

// Mutability
s[0] = 'J'; // "Jello World"

// Substring
std::string sub = s.substr(0, 5); // "Jello"

// Length
int len = s.length(); // or s.size()

// Find
size_t pos = s.find("World");

// Reverse (In-place)
std::reverse(s.begin(), s.end());

String (Immutable) & StringBuilder

In Java, String is completely immutable. Modifying a string creates a new string. For heavy manipulation (loops, reversing), you MUST use StringBuilder.

String s = "Hello World";

// Length
int len = s.length();

// Access character (No [] operator!)
char c = s.charAt(0); // 'H'

// Substring (startIndex, endIndexExclusive)
String sub = s.substring(0, 5); // "Hello"

// Split
String[] parts = s.split(" "); // ["Hello", "World"]

// -------------------------------------------
// STRINGBUILDER (For mutability & reversing)
// -------------------------------------------
StringBuilder sb = new StringBuilder(s);
sb.setCharAt(0, 'J');    // Modifies in-place
sb.reverse();            // Reverses in-place
String newString = sb.toString();

str (Immutable)

Python strings are immutable. You cannot change a character by index. Python excels at string manipulation through its incredibly powerful slicing syntax.

s = "Hello World"

# Length
length = len(s)

# Immutability Check
# s[0] = 'J' # TypeError: 'str' object does not support item assignment

# Mutating workaround: Replace or Slicing
s = "J" + s[1:] # "Jello World"

# Slicing (startIndex : endIndexExclusive)
sub = s[0:5] # "Jello"

# Split by delimiter
parts = s.split(" ") # ['Jello', 'World']

# Reverse a string (Idiomatic Python Slicing)
reversed_s = s[::-1]

# Join array back to string
joined = "-".join(parts) # "Jello-World"

String (Immutable) & Template Literals

JavaScript strings are immutable. JS features powerful Template Literals (backticks) for interpolation. Reversing a string requires splitting it into an array first.

let s = "Hello World";

// Length
let len = s.length;

// Immutability Check
s[0] = 'J'; // Fails silently, string remains "Hello World"

// Template Literals (Interpolation)
let name = "Rakesh";
let greeting = \`Hello \${name}, welcome!\`;

// Substring
let sub = s.substring(0, 5); // "Hello"

// Split
let parts = s.split(" "); // ["Hello", "World"]

// Reverse a string (Split to Array -> Reverse -> Join)
let reversed = s.split("").reverse().join("");

// Mutating workaround
s = "J" + s.slice(1); // "Jello World"
