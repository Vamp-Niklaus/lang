Algorithms (<algorithm>)

A comprehensive suite of functions operating on ranges of elements via iterators.

Category: Core Tools

Query Algorithms (Non-Mutating)

for_each(first, last, f)
Applies f to each element.
find(first, last, val)
Finds first occurrence of val.
find_if(first, last, pred)
Finds first element matching pred.
adjacent_find(first, last)
Finds first adjacent duplicate.
count(first, last, val, &n)
Counts occurrences of val.
mismatch(f1, l1, f2)
Finds first mismatched pair.
equal(f1, l1, f2)
Checks if ranges are equal.
search(f1, l1, f2, l2)
Finds subsequence.

Mutating Algorithms

copy(f1, l1, dest)
Copies elements to dest.
swap(x, y)
Swaps two values.
transform(f1, l1, dest, op)
Applies op to range, outputs to dest.
replace(f, l, old, new)
Replaces old with new.
fill(f, l, val)
Fills range with val.
generate(f, l, gen)
Assigns result of gen() to range.
remove(f, l, val)
Shifts to hide val (must follow with erase).
unique(f, l)
Removes consecutive duplicates.
reverse(f, l)
Reverses elements.
rotate(f, mid, l)
Left rotates by mid-first.
random_shuffle(f, l)
Randomly permutes range.
partition(f, l, pred)
Groups true pred elements first.

Sort & Binary Search (Requires Sorted)

sort(f, l)
O(N log N) sort.
stable_sort(f, l)
Sort preserving relative order.
partial_sort(f, m, l)
Sorts up to m.
nth_element(f, n, l)
Places nth element in sorted pos.
binary_search(f, l, val)
Returns true if found.
lower_bound(f, l, val)
First iter >= val.
upper_bound(f, l, val)
First iter > val.
equal_range(f, l, val)
Pair of lower/upper bounds.
merge(f1, l1, f2, l2, res)
Merges 2 sorted ranges.

Set Operations & Heaps

set_union(f1, l1, f2, l2, r)
set_intersection(f1, l1, f2, l2, r)
set_difference(...)
set_symmetric_difference(...)
make_heap(f, l)
push_heap(f, l)
pop_heap(f, l)
sort_heap(f, l)

Min/Max & Permutations

min(a, b) / max(a, b)
min_element(f, l) / max_element(f, l)
next_permutation(f, l)
prev_permutation(f, l)
lexicographical_compare(f1,l1,f2,l2)

Collections & Streams

Java provides `java.util.Collections` for standard algorithms like sorting and searching. Java 8 Streams provide equivalents to `std::transform`, `std::count_if`, etc.

import java.util.Collections;
import java.util.Arrays;

// Sort
Collections.sort(list);
Arrays.sort(arr);

// Binary Search
int idx = Collections.binarySearch(list, 10);

// Reverse
Collections.reverse(list);

Built-ins & itertools

Python relies heavily on built-in functions (`sorted()`, `reversed()`, `max()`) and the `itertools` module.

import bisect

# Sort
lst.sort()       # In-place
new_lst = sorted(lst) # Returns new list

# Binary Search
idx = bisect.bisect_left(lst, 10)

# Reverse
lst.reverse()

Array.prototype

JavaScript attaches algorithm equivalents directly to the Array prototype.

// Sort (Requires comparator for numbers!)
arr.sort((a, b) => a - b);

// Find
let val = arr.find(x => x > 10);

// Reverse
arr.reverse();
