List (std::list)

Doubly-linked list. Fast insertions and deletions anywhere.

Category: Sequence Container

Core Operations

#include <list>
std::list<int> l;

l.push_back(10);
l.push_front(5);
l.pop_back();
l.pop_front();

l.insert(l.begin(), 20);
l.erase(l.begin());

List-Specific Operations

l.splice(pos, other_list);    // Move elements from another list
l.remove(val);                // Remove all elements matching val
l.remove_if(pred);            // Remove elements matching predicate
l.unique();                   // Remove consecutive duplicates
l.merge(other_list);          // Merge two sorted lists
l.sort();                     // O(N log N) sort
l.reverse();                  // Reverse the list

LinkedList<T>

java.util.LinkedList is a pure doubly-linked list implementation. Elements are not stored continuously, meaning memory overhead per node is higher, but insertions/deletions in the middle of the list via Iterator are O(1).

import java.util.LinkedList;
import java.util.ListIterator;

LinkedList<Integer> list = new LinkedList<>();
list.addLast(20);
list.addFirst(10);

// O(n) Access - Unlike ArrayDeque, indexing requires traversal
int val = list.get(1);

// O(1) insertion via iterator
ListIterator<Integer> it = list.listIterator();
if (it.hasNext()) {
    it.next();
    it.add(15); // Inserts right after current node
}

collections.deque

Python does not have a strict doubly-linked list standard library class that exposes internal node manipulation. collections.deque is the closest equivalent and provides O(1) front/back insertions.

from collections import deque

lst = deque()
lst.append(10)      # back
lst.appendleft(5)   # front

# Middle insertion is O(n) because Python deque 
# doesn't expose node iterators directly.
lst.insert(1, 7)

Custom Implementation

JavaScript has no built-in linked list. If you truly need O(1) middle insertions, you must write a custom Node class.

class Node {
    constructor(val) {
        this.val = val;
        this.prev = null;
        this.next = null;
    }
}
